# Release Example

## Scenario

A project needs to publish a routine release after a small batch of changes.

## Applied Capability

- `skills/release/release-management/SKILL.md`
- `skills/testing/test-strategy/SKILL.md`
- `workflows/release_workflow.md`

## Result

The release is prepared with explicit scope, validation steps, and rollback notes before it is shipped.

## Notes

- Validate the release candidate first.
- Keep the release notes concise.
- Confirm post-release health after deployment.
