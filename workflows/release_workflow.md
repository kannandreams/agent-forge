# Release Workflow

## Purpose

Prepare and ship a release in a controlled, repeatable way.

## Sequence

1. Confirm release scope.
2. Review linked work and open risks.
3. Update release notes or changelog.
4. Validate the build or package.
5. Execute the release action.
6. Confirm the release outcome.
7. Record any follow-up work.

## Rules

- Ship only what is intended for the release.
- Verify before publishing.
- Keep release steps explicit and repeatable.
- Treat rollback or mitigation as part of the workflow, not an afterthought.
