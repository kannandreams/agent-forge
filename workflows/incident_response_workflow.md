# Incident Response Workflow

## Purpose

Coordinate an incident response path from detection to follow-up.

## Sequence

1. Use the incident response skill to confirm scope and severity.
2. Stabilize the affected system or user path.
3. Gather the minimal facts needed to make the next decision.
4. Decide whether mitigation, rollback, or escalation is required.
5. Record the timeline and key actions.
6. Create follow-up items for investigation and prevention.

## Rules

- Prioritize containment.
- Keep communication factual.
- Record decisions as they happen.
- Avoid speculative root-cause claims during active response.

## Expected Result

A response should end with:

- a stabilized incident state or an explicit escalation path
- a short timeline of what happened
- tracked follow-up work
- enough context for a later postmortem
