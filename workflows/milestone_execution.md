# Milestone Execution Workflow

## Purpose

Execute a project milestone from planning through delivery using reusable skills and templates.

## Sequence

1. Use the milestone skill to define the milestone scope and acceptance criteria.
2. Use the issue breakdown skill to split work into actionable tasks.
3. Use the milestone template to standardize the execution plan.
4. Coordinate implementation work and track dependencies.
5. Validate the result against the acceptance criteria.
6. Record outcomes, risks, and follow-up items.

## Rules

- Keep the milestone bounded.
- Prefer clear ownership.
- Verify each step before moving to the next one.
- Do not mix planning output with implementation output.

## Expected Result

A milestone should end with:

- completed or clearly tracked work items
- validated acceptance criteria
- documented follow-up items
- a reusable record for future milestones
