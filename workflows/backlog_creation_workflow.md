# Backlog Creation Workflow

## Purpose

Turn a project goal into a lightweight backlog of actionable items.

Use this workflow when the team needs a fast planning pass and does not need a full milestone plan.

## Sequence

1. Use the backlog creation skill to frame the goal and capture context.
2. Identify the smallest useful set of backlog items.
3. Order the items by dependency and priority.
4. Add short notes for each item, including assumptions or open questions.
5. Confirm the backlog is actionable and not over-specified.

## Rules

- Keep the output small and usable.
- Avoid turning backlog creation into full design.
- Prefer items that can be picked up independently.
- Capture dependencies only when they affect execution order.

## Expected Result

A backlog should end with:

- a short list of actionable items
- rough sequencing
- brief dependency notes
- any open questions that still need resolution
