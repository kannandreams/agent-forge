## Commit Type Notes

Use these notes as input for a future skill. They are not yet repository policy.

| Type | Use When |
|------|----------|
| `feat` | User-visible behavior or production code capability changes. |
| `fix` | A bug is corrected; a failing reproduction would pass after the change. |
| `refactor` | Production code changes without intended behavior change. |
| `perf` | Production code changes improve a measured performance characteristic. |
| `style` | Formatting or whitespace changes only. |
| `docs` | Markdown or comments change without runtime behavior changes. |
| `test` | Test files or fixtures change. |
| `ci` | CI configuration or workflow behavior changes. |
| `build` | Build configuration, packaging, or dependency resolution changes. |
| `chore` | Maintenance that does not fit another type and does not affect source, tests, docs, CI, or build behavior. |
| `revert` | A previous commit is intentionally reverted. |
